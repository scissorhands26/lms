# Network Administration

## What is a Terminal

A terminal is an essential tool for system and network administrators, offering a text-based interface to interact with the computer.

- **Interface for Command Execution**: A terminal is a text-based interface that allows users to execute commands on their computer.
- **Direct System Interaction**: Provides direct interaction with the operating system, enabling tasks like file management, process control, and network configuration.
- **Essential for Administrators**: Widely used by system and network administrators for efficient and precise system management.

## What is a Command

Understanding commands is fundamental for anyone involved in network administration and system management.

- **Instructions for the Computer**: Commands are specific instructions given to a computer to perform a particular task.
- **Text-Based Input**: Typically entered through a command-line interface (CLI) or terminal.
- **Direct Interaction**: Allows direct interaction with the operating system to manage files, processes, and network settings.
- **Essential for Administration**: Crucial for system and network administration, troubleshooting, and automation tasks.

## What is a Command Option

Command options, also known as flags or switches, modify the behavior of commands to provide additional functionality.

- **Definition**: Options (or flags) modify the behavior of commands.
- **Syntax**: Typically follow the command, prefixed with a dash (-) or double dash (--).
- **Examples**:
  - `ls -l`: Lists files in long format.
  - `ping -c 4 <IP>`: Sends 4 packets to the specified IP.
- **Combining Options**: Multiple options can be combined (e.g., `ls -la`).

## What is a Command Argument

Command arguments are additional information provided to a command to specify the target or operation to be performed.

- **Definition**: Arguments provide additional information to a command.
- **Positional**: Arguments are typically placed after the command and options.
- **Examples**:
  - `cp file1 file2`: Copies `file1` to `file2`.
  - `mkdir directory`: Creates a new directory named `directory`.
- **Required vs. Optional**: Some arguments are required, while others are optional.

## What is a Command Prompt

A command prompt is a text-based interface that allows users to interact with the operating system by entering commands.

- **Text-Based Interface**: A command prompt provides a text-based interface for users to interact with the operating system.
- **Input and Output**: Users can enter commands and receive output directly in the terminal.
- **Customization**: Command prompts can be customized with different colors, fonts, and additional information.
- **Common Prompts**:
  - **Linux**: `$` for regular users, `#` for root (superuser).
  - **Windows**: `C:\>` for regular users, `C:\WINDOWS>` for administrators.

## What is a Shell

A shell is a command-line interface that interprets user commands and executes them on the operating system.

- **Command Interpreter**: A shell interprets user commands and executes them on the operating system.
- **Types of Shells**:
  - **Bourne Shell (sh)**: The original Unix shell.
  - **Bash (Bourne Again Shell)**: The default shell on most Linux distributions.
  - **Zsh (Z Shell)**: An extended version of Bash with additional features.
  - **PowerShell**: The default shell on Windows systems.
- **Scripting**: Shells can be used for scripting to automate tasks.

## What is a Script

A script is a file containing a sequence of commands that can be executed by a shell or interpreter.

- **Sequence of Commands**: A script contains a sequence of commands that can be executed by a shell or interpreter.
- **Automation**: Scripts are used to automate repetitive tasks and complex operations.
- **File Extension**: Scripts are typically saved with a `.sh` extension for shell scripts or `.ps1` for PowerShell scripts.
- **Interpreted**: Scripts are interpreted by the shell or scripting engine, rather than compiled like traditional programs.

## Command List

Commands are essential for managing and troubleshooting network systems. The table below compares key Linux and Windows commands for processes, connections, and settings. Mastering these commands ensures efficient network management.

| **Description**          | **Linux**                | **Windows (CLI)**                                    | **Windows (PowerShell)**                                                    |
| ------------------------ | ------------------------ | ---------------------------------------------------- | --------------------------------------------------------------------------- |
| List active sockets      | `ss -natup`              | `netstat -ano`                                       | `Get-NetTCPConnection`                                                      |
| List listening ports     | `ss -lntup`              | `netstat -an find "LISTENING"`                       | `Get-NetTCPConnection -State Listen`                                        |
| Check network interfaces | `ip addr show`           | `ipconfig`                                           | `Get-NetAdapter`                                                            |
| Display ARP table        | `arp -a`                 | `arp -a`                                             | `Get-NetNeighbor`                                                           |
| Display routing table    | `route -n`               | `route print`                                        | `Get-NetRoute`                                                              |
| Display hostname         | `hostname`               | `hostname`                                           | `Get-ComputerInfo`                                                          |
| Connect to port          | `nc -vz <IP> <PORT>`     | `Test-NetConnection -ComputerName <IP> -Port <PORT>` | `Test-NetConnection -ComputerName <IP> -Port <PORT>`                        |
| Capture network packets  | `tcpdump -i <INTERFACE>` | `netsh trace start capture=yes`                      | `New-NetEventSession -CaptureMode Local -LocalFilePath C:\temp\capture.etl` |
